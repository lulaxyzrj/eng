# Lesson 01: Your First Bundle

This lesson introduces Databricks Asset Bundles using the official template generated by `databricks bundle init`.

---

## 🎯 Learning Objectives

- Create a bundle from the official Databricks template
- Understand the standard bundle structure
- Deploy your first bundle to a workspace
- Observe deployed resources in the Databricks UI

---

## How This Bundle Was Created

```bash
databricks bundle init
```

The CLI prompted for:
- Template selection (default Python)
- Bundle name
- Initial configuration

This generated the complete folder structure you see here.

---

## Key Configuration Files

### `databricks.yml`

```yaml
bundle:
  name: bundle_databricks_exemple
  uuid: f193bf07-9497-4aa8-ba09-0157f4887eec

include:
  - resources/*.yml
  - resources/*/*.yml

artifacts:
  python_artifact:
    type: whl
    build: uv build --wheel

variables:
  catalog:
    description: The catalog to use
  schema:
    description: The schema to use

targets:
  dev:
    mode: development
    default: true
    workspace:
      host: https://adb-xxx.azuredatabricks.net  # ← Update this!
    variables:
      catalog: ubereats_dev
      schema: bronze
```

### Key Elements Explained

| Element | Purpose |
|---------|---------|
| `bundle.name` | Unique identifier for this bundle |
| `include` | Glob patterns to load resource definitions |
| `artifacts` | Build artifacts (wheels) to deploy |
| `variables` | Parameterized values for reuse |
| `targets` | Environment-specific configurations |

---

## ⚠️ Required Configuration

Before deploying, update `databricks.yml`:

```yaml
targets:
  dev:
    workspace:
      host: https://YOUR-WORKSPACE.azuredatabricks.net  # ← Your URL
    variables:
      catalog: YOUR_CATALOG  # ← Your Unity Catalog name
```

## Deployment Steps

### 1. Validate Configuration

```bash
cd 01-bundle-databricks-exemple
databricks bundle validate -t dev
```

Expected output: No errors, configuration is valid.

### 2. Deploy to Workspace

```bash
databricks bundle deploy -t dev
```

This will:
- Build the Python wheel
- Upload code to workspace
- Create/update the DLT pipeline
- Create/update the job

### 3. Verify in UI

```bash
databricks bundle summary -t dev
```

Or open your workspace and navigate to:
- **Workflows** → Look for `[dev your_name] sample_job`
- **Delta Live Tables** → Look for your pipeline

### 4. Run the Pipeline (Optional)

```bash
databricks bundle run -t dev bundle_databricks_exemple_etl
```

---

## Understanding `mode: development`

Because this target uses `mode: development`:

- Resources are prefixed with `[dev your_username]`
- Schedules are created in **PAUSED** state
- Perfect for safe experimentation

👉 **[Read more about modes](../doc_utils/DABS-MODES-EXPLAINED.md)**

---

## Clean Up

When you're done experimenting:

```bash
databricks bundle destroy -t dev
```

This removes all deployed resources.
